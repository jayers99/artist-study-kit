# Source Hierarchy and Integration Strategy for Automated Art Retrieval

## 1. Executive Tiering: The Source Hierarchy

To architect a scalable automated art retrieval pipeline, sources must be tiered based on metadata authority, license reliability, and technical throughput. This hierarchy prioritizes institutional "ground truth" over third-party aggregators.

### 1.1 Tier 1: Pivot & Identity Layer (Wikidata)
Wikidata serves as the "master connector" and primary identity provider. It bridges disparate institutional IDs (Met ObjectIDs, AIC ArtistIDs, etc.) using its own QIDs as the universal primary key for cross-source deduplication.
*   **SPARQL Querying:** Use the Wikidata Query Service to find works via the `creator (P170)` property. 
*   **Identity Resolution:** The QID allows the pipeline to unify records for a single artist or work found across different museum APIs, enabling precise metadata enrichment and preventing redundant ingestion.

### 1.2 Tier 2: Primary Institutional Authorities (The "Big Three")
These sources provide the highest resolution assets and machine-readable rights metadata. 

| Rank | Institution | Coverage | API Reliability & Features | Licensing Clarity |
| :--- | :--- | :--- | :--- | :--- |
| 1 | **The Metropolitan Museum of Art** | 500,000+ objects | **High.** MCP wrapper supports parallel batch-fetching (20 IDs/call). | **Excellent.** Native `isPublicDomain` filter for CC0. |
| 2 | **The Art Institute of Chicago (AIC)** | High | **Very High.** Detailed REST API; Nightly GitHub Data Dumps for bulk sets (>10k). | **Strong.** Robust `is_public_domain` boolean. |
| 3 | **The Cleveland Museum of Art (CMA)** | 68,000+ objects | **Moderate.** No-key REST access; requires 300ms polite delay. | **Excellent.** Explicit `share_license_status` metadata. |

### 1.3 Tier 3: Aggregators and Global Repositories (Wikimedia Commons & Europeana)
Aggregators are utilized for broad discovery when institutional IDs are missing.
*   **Structured Data on Commons (SDC):** The pipeline leverages the transition to machine-readable SDC. High-accuracy resolution is achieved through a triple-property check: `depicts (P180)`, `main subject (P921)`, and `digital representation of (P6243)`.

### 1.4 Tier 4: Secondary Discovery (DPLA, Google Arts & Culture, WikiArt)
These layers are designated "Thumbnail-only." They lack reliable, machine-readable rights management and should be used only for initial curation boards. Any work discovered here must be verified through Tier 2 or Tier 1 before high-resolution resolution.

## 2. Per-Source Technical Profiles

### 2.1 The Art Institute of Chicago (AIC)
The AIC offers both a REST-style API and nightly data dumps, ideal for varying integration scales.
*   **Data Acquisition:** Use the **AIC Nightly Data Dumps** (JSON files via GitHub) for initial large-scale ingestion (>10,000 records). Use the REST API for real-time verification and detail retrieval.
*   **IIIF Implementation:** Construct image URLs using the pattern `[config.iiif_url]/[image_id]/full/[width],/0/default.jpg`. 
    *   Use `843` width for standard/copyrighted thumbnails.
    *   Use `1686` width for CC0/Public Domain works (confirmed via `is_public_domain`).
*   **Rate Limiting:** Unauthenticated API requests are throttled at **60 requests per minute**. Scraping logic must include a **1-second delay** between image downloads to maintain stability.

### 2.2 The Metropolitan Museum of Art (The Met)
The Met's data can be accessed via their native REST API (`collectionapi.metmuseum.org`) or a third-party **Model Context Protocol (MCP) server** for enhanced batch performance.
*   **Search Filters:** The pipeline must utilize search parameters `isPublicDomain`, `hasImages`, `isHighlight`, `isOnView`, and `medium` for precision.
*   **Batch Normalization:** The MCP `met_get_object` tool enables parallel fetching of up to 20 objects. These records return **Getty ULAN URLs** and **AAT IDs**, providing secondary normalization pivots for artist and classification metadata when Wikidata QIDs are unavailable.

### 2.3 Cleveland Museum of Art (CMA)
CMA provides an "Open Access" API with no authentication requirement, facilitating rapid integration.
*   **Serialized Metadata Parsing:** Critically, CMA serializes several key fields as JSON strings. The pipeline must execute `JSON.parse()` on the `creators`, `culture`, `provenance`, and `exhibitions` fields to access the underlying data arrays.
*   **License Verification:** The `share_license_status` field must be monitored for the "CC0" string to unlock high-resolution image links.

### 2.4 Wikimedia Commons & SDC
The system utilizes Structured Data on Commons (SDC) to bridge media files with the Wikidata knowledge graph.
*   **High-Accuracy Resolution:** The pipeline prioritizes images where the `digital representation of (P6243)` property is present. For maximum accuracy, the system cross-references `P6243` against the museum's local artist ID or the Wikidata QID to ensure the file is an authoritative representation of the specific 2D artwork.

## 3. The Combination & Deduplication Strategy

The integration pipeline executes a logical flow to resolve data conflicts and ensure unique results.

1.  **Discovery (Broad Search):** Perform keyword-based queries across Tier 3 and Tier 4 to populate initial "Curation Boards."
2.  **Attribution Verification (The Pivot):** Resolve the work to a **Wikidata QID**. This QID serves as the unique identifier used to back-reference the work to an official Tier 2 institutional record. 
3.  **Deduplication & Conflict Resolution:** If multiple sources provide the same work, the Tier 2 Institutional record always supersedes Tier 3 or 4 data. 
    *   *Logic:* If `Wikimedia_Commons.P6243 == Institutional.ID` (mapped via Wikidata), the institutional metadata (including Getty ULAN/AAT data) is treated as the primary source.
4.  **Resolution to High-Resolution:** High-res downloads are only triggered after a multi-institutional license check confirms CC0 status.

## 4. Copyright & Legal Posture

The pipeline is governed by a "Safe Harbor" operating framework to maintain institutional compliance.

*   **Thumbnail Usage:** For copyrighted or unverified material, the pipeline strictly adheres to the 843px width limitation, a field standard for non-CC0 private curation.
*   **Mandatory Licensing Checklist:** A high-resolution download (IIIF or direct) is prohibited unless the following metadata values are present and verified:
    *   **The Met:** `isPublicDomain` must be `true`.
    *   **AIC:** `is_public_domain` must be `true`.
    *   **CMA:** `share_license_status` must equal `"CC0"`.
*   **Forbidden Resolution Sources:** Aggregators such as WikiArt and general search APIs (Google/Bing) are restricted to **Discovery Only**. These platforms do not provide reliable rights-management metadata and cannot be used to verify "Safe Harbor" status for high-resolution assets.
