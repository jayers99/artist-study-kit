# Worked Examples, Aids & Fading

How to design study aids that build mastery and then disappear. Source:
[[07.1-study-aids-scaffolding]]. Used by [[stage-study-retention]] (drills, cheat sheets,
checklists), [[stage-style-definition]] / [[stage-visual-analysis]] (the technique-imitation
checklist as a job aid), and [[stage-curation-interview]] (the attempt-first study brief).
Pairs with [[concept-desirable-difficulty]].

## Job aid vs learning aid

- **Job aid (performance support):** permanent "external memory" for task execution.
- **Learning aid (instructional):** *temporary* — builds schema, then is **faded**.
A study tool must be clear which it is building; conflating them creates crutches.

## Cognitive architecture

Working memory holds ~4–7 chunks. Reduce **extraneous load** (clutter), manage **intrinsic
load** (chunking), foster **germane load** (the productive schema-building effort). Respect
the dual-channel assumption; avoid split-attention and redundancy effects.

## Worked examples → fading

The **worked-example effect** cuts wasteful problem-search for novices. Best practices:
isolate building blocks, show relations, use completion problems, subgoal labeling,
self-explanation prompts. Then **fade** steps to avoid the **expertise-reversal effect**.

## Checklists

Cognitive safety nets that externalize the error search and liberate System 2 — used by
experts, not just novices (working memory is fragile under load).

## The crutch problem

Cognitive offloading (esp. to AI) bypasses productive struggle; one study found r = -0.688
between AI dependency and cognitive ability. Counter with the **attempt-first protocol**
(struggle before consulting the aid) and, for high-prior-knowledge learners, have *them*
build the aid (SOI: select-organize-integrate).

## Art-study aid templates

Style cheat sheet (chunked "structural DNA") · value/composition checklist (error
prevention) · technique reference card (numbered steps) · "what to notice first" prompt
(interrupts System 1).

## Fading in the study brief

The [[stage-curation-interview]] output is a faded worked example: each brief's `study_plan`
is an ordered protocol with **success tests** that strips support as skill grows — e.g.
monochrome line only → add ~4 grays → add color chords last, each step testable ("trace your
eye-path — does it loop?"). Merged study→final pairs (a prep drawing then its painting) are a
built-in completion-problem progression. This is the worked-example effect applied to copying:
isolate one rule, fade toward independent variation.
